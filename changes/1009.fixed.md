The recommended unique index on the `Outbox` table is now composite over (`message_id`, `target_broker`) instead of `message_id` alone. A single published event is dual-written to the outbox once per target broker (the internal broker plus every external broker), so a unique index on `message_id` alone rejected the framework's own multi-broker dual-write with a `UniqueViolation`. Existing deployments that already created the `uq_outbox_message_id` index should drop it and recreate the composite index (see the v0.16 migration guide).
