Add Kubernetes-compatible health check endpoints for Engine and FastAPI applications. The Engine embeds a lightweight HTTP server with `/healthz` (liveness), `/livez` (alias), and `/readyz` (readiness) endpoints. For FastAPI apps, `create_health_router(domain)` provides equivalent endpoints as a reusable router. Readiness checks cover database providers, brokers, event store, caches, and subscription count. Configure via `[server.health]` in `domain.toml` (enabled by default on port 8080).

**Default behaviour change — Tier 2.** `protean server` now binds port 8080 on startup. The engine logs a warning and continues without probes if the port is already in use, so the change is non-fatal — but operators running another service on 8080 will want to either disable the health server (`[server.health] enabled = false`) or move it (`[server.health] port = ...`) before upgrading. The design rationale is captured in [ADR-0012](docs/adr/0012-health-check-architecture.md).
