Fixed `identity_type = "uuid"` producing a native `uuid.UUID` on the auto-injected `id` field (and on nested-entity ids), which is not JSON-serializable and so broke `to_dict()`, event/command payloads, and API responses. A uuid identity is now a UUID *string* in Python on every path, consistent with an explicitly declared `Auto(identity_type="uuid")` field; adapters that support a native UUID column (e.g. SQLAlchemy's `GUID` type) still store it as one. Integer identities are unaffected. See ADR-0021 and the 0.17 upgrade notes.
