Event store **category** and **`$all`** reads now page by `global_position` (inclusive) consistently across the memory and MessageDB adapters. The memory adapter previously filtered and ordered these reads by the per-stream ordinal, so a paginated read (any `position > 0`) returned the wrong batch — often nothing: always for a multi-stream category, and even for a single stream once a subscription advanced its cursor and re-read from the next position. The MessageDB `$all` read additionally skipped its first position and returned an unordered page; it now reads `global_position >= position` in ascending order. Specific-stream reads (`category-id`) still page by their per-stream position.
