Add an injectable `domain.clock` seam so tests can make time-dependent behavior deterministic. Every domain now exposes `domain.clock` (a `protean.utils.Clock`), defaulting to `SystemClock` (real UTC), and the command-deadline path, retry/lock boundaries on the `Outbox` aggregate, and message timestamps read "now" through it *when a domain context is active* (outside a context they fall back to real UTC, so plain scripts and pre-bootstrap workers are unaffected). Assign a stub clock (`domain.clock = my_clock`) within a pushed context to freeze or advance time and prove exact-instant deadline/expiry/backoff semantics without monkeypatching `datetime`. Default (`SystemClock`) behavior is unchanged, with one benign refinement: each outbox `mark_*`/`start_processing` call now snapshots a single "now" and reuses it for every field it writes, so timestamps that previously differed by microseconds within one call are now identical. The `protean new` example scaffold now generates timezone-aware `datetime.now(UTC)` timestamps instead of naive `datetime.now()`.
