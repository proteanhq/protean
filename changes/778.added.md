Add three info-level naming-convention diagnostics to the IR builder: `EVENT_NOT_PAST_TENSE` flags domain events whose name ends in `-ing` (`OrderPlacing`), `COMMAND_NOT_IMPERATIVE` flags commands that do not begin with a verb from the pinned `IMPERATIVE_VERBS` set (`OrderCreation`), and `AGGREGATE_NOT_NOUN` flags aggregates whose name ends in a verb, gerund, or adjective suffix (`OrderProcessing`, `Notify`, `Recursive`). All three are heuristic suffix/prefix checks — advisory (`level: "info"`), so they never gate CI under the default `[lint].level = "warn"` floor, and they pass through the existing per-element `suppress_checks` / `[lint].suppressions` channels.
