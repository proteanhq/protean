Lowered the `>=` version floors on runtime dependencies (fastapi, typer, uvicorn, pydantic, sqlalchemy, structlog, marshmallow, and others) to the oldest versions Protean actually supports, widening the range of versions an application can hold while depending on Protean. Floors that encode a real constraint are kept: fastapi 0.139 (Starlette-1.x routing), elasticsearch 8.18 (bundled `elasticsearch.dsl`), jinja2 3.1.6 (CVE-2025-27516), and the C-extension packages at their first Python 3.14 wheel. A new CI leg proves the floors on every run by resolving them with `uv --resolution lowest-direct`, and Dependabot now uses `increase-if-necessary` so it no longer ratchets floors up to the newest release. See ADR-0020.
