Added `protean.testing.process_and_wait` and `protean.testing.drain` — integration-test helpers that run a command through its processing pipeline and settle the result, so the same test body works in both processing modes: inline during the call under synchronous processing, or by draining a bounded test-mode engine (outbox → broker → subscription → projector) under asynchronous processing. The returned `ProcessResult` exposes the command result, the events that fired (captured via the command's correlation chain), and any synchronous/submission-time error — without the caller reaching into framework internals.
