`protean idempotency cleanup` prunes consume-side idempotency markers (`ProcessedMessage`) older than a retention window, in bounded batches, so the marker table's growth can be bounded (run it periodically, e.g. from cron — it is not auto-scheduled). The retention window and batch size are configurable via `[consume_idempotency.cleanup]` (default 7 days / 5000 rows). See ADR-0017.
