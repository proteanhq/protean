The `protean check` exit code now honours a `[lint].level` config key (default `"warn"`). `"error"` fails CI only on errors, `"warn"` fails on errors and warnings, and `"info"` fails on any error, warning, or info finding. Errors always exit `1`; a gating non-error finding exits `2`. The default reproduces the previous behaviour exactly, and the `--level` CLI flag remains display-only.
