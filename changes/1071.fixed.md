The in-memory repository now enforces `Index(unique=True)` declarations. A save or update that violates a single-column or composite unique index raises `ValidationError` (the same Protean-level error already raised for field-level `unique=True`), so memory-mode tests catch duplicate-key regressions instead of silently accepting them. Relational backends continue to reject the duplicate at the database. NULLs are treated as distinct, matching PostgreSQL/SQLite semantics.
