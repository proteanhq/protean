Fixed Elasticsearch database setup crashing with `KeyError` when the domain registers a cache-backed projection (`@domain.projection(cache=...)`) — the Elasticsearch analogue of the SQLAlchemy break fixed in #1034. Provider table/index ownership is now decided by a single `BaseProvider.owns()` gate, which also excludes entities nested at any depth under an event-sourced aggregate and raises `ConfigurationError` when an element references an unknown provider.
