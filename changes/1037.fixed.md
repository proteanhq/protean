Elasticsearch database setup no longer crashes when a domain registers a cache-backed projection (`@domain.projection(cache=...)`). The provider's index create/drop/reset loops previously resolved `providers[cls.meta_.provider]` unconditionally, so a cache-backed projection (`meta_.provider is None`) made `providers[None]` raise `KeyError` — an analogue of the cache-backed-projection break #1034 fixed for SQLAlchemy. Provider table/index ownership is now decided by a single `BaseProvider.owns()` gate shared across providers. It resolves an element's root aggregate via `meta_.aggregate_cluster`, so entities nested under an event-sourced aggregate are correctly excluded at any depth (a single-hop `part_of` check previously missed 2nd-level entities), and it compares provider names rather than indexing on a `None` provider.
