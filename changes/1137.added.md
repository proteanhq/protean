`protean schema generate` now emits **Avro** and **Protobuf** schemas in addition to JSON Schema, via `--format avro|protobuf|json` (default `json`). Per-version files are written as `{name}.v{version}.avsc` / `.proto`. Avro records use null-first unions for optionals and logical types for identifier/date/datetime; Protobuf messages use proto3 `optional`/`repeated`, well-known types, and field numbers assigned deterministically in sorted field-name order, so re-generating an *unchanged* schema is byte-identical. (Adding a field renumbers alphabetically-later fields — the numbers are for schema emission, not wire-compatible evolution.)
