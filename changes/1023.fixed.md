Fix `@handle("$any")` event handlers never firing under `event_processing="sync"`. The synchronous dispatch path routes through `EventStore.handlers_for`, which matched only the concrete event `__type__` and silently dropped handlers keyed under `"$any"`, so a wildcard handler (and any read model or side effect it drives) never ran in sync mode — though it fired correctly under async (Engine) dispatch. `handlers_for` now also returns handlers that declare a `$any` handler, on both the internal- and external-event paths.
