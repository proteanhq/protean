Add per-subscription OTEL metrics for message processing. New counters: `protean.subscription.messages_processed` (with status ok/error), `protean.subscription.retries`, and `protean.subscription.dlq_routed` — all with `{subscription, handler, stream}` attributes. New histogram: `protean.subscription.processing_duration` for per-message latency. New engine observable gauges: `protean.engine.up`, `protean.engine.uptime_seconds`, and `protean.engine.active_subscriptions`. All metrics are no-op when OpenTelemetry is not installed.
