`protean upgrade-check` — a read-only CLI diagnostic that inspects a domain (and its live database schema) and reports the changes needing attention when upgrading to 0.16: status fields that may reject self-transitions, raised SQLAlchemy pool defaults, the health-check port binding, the Elasticsearch v8-client/server requirement, and unbounded outbox columns. For the outbox it generates the exact backend `ALTER` SQL to review and run; nothing is applied automatically.
