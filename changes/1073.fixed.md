Fixed `reconcile_outbox` (the ADR-0015 crash-window recovery) being a permanent no-op with the Message-DB event store. `read_last_message("$all")` returned `None` for category streams, so reconciliation short-circuited before scanning and a durable-but-unpublished event was never repaired. The Message-DB adapter now resolves `$all`/category streams (falling back to a category read, picking the newest by global position), so crash-window recovery and the engine's startup sweep work on Message-DB-backed domains.
