The core test suite no longer leaks file descriptors when run as a single process: event loops and Starlette/FastAPI `TestClient` instances created during a test are now closed at teardown, so `protean test` runs under the default 1024 open-file limit without `OSError: Too many open files`. A CI gate enforces this at `ulimit -n 1024`, and `PROTEAN_FD_REPORT=1` logs per-module descriptor deltas to locate future leaks.
