Fix `protean server --workers N` silently double-processing events on the event-store subscription path. Event-store subscriptions read directly from the event store with no cluster-wide ownership, so every worker processes the same events. The server now refuses to start more than one worker when any handler resolves to an event-store subscription, naming the offending handlers and pointing to the three ways forward: run a single worker, switch those handlers to stream subscriptions (`subscription_type = "stream"`), or pass the new `--acknowledge-event-store-risk` flag to override. The guard is enforced at both the `protean server` CLI and the programmatic `Supervisor` entry point (which gains an `acknowledge_event_store_risk` parameter). Stream-based and broker-only deployments are unaffected and continue to scale horizontally.
