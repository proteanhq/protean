`protean ir check` now distinguishes a baseline built against a different IR schema version from stale content: when the stored `ir_version` differs from the current schema version it reports a version mismatch (exit code 3, with `stored_version`/`current_version` in JSON output) instead of a generic stale result. A new `tests/ir/test_schema_publication.py` guards that the published schema under `docs/assets/ir/vX/` stays byte-identical to the runtime schema under `src/protean/ir/schema/vX/`, and the previously drifted `v0.1.0` published copy has been re-synced.
