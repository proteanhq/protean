Add opt-in tail sampling for wide events via `[logging.sampling]` in `domain.toml`. New `TailSamplingProcessor` (structlog) and `TailSamplingFilter` (stdlib) make keep/drop decisions on `protean.access` events based on outcome — errors (`status="failed"` or level >= ERROR), slow requests (`status="slow"`), and message types matching `critical_streams` glob patterns are always kept; everything else is sampled at `default_rate` (5% by default). Kept events carry `sampling_decision`, `sampling_rule`, and `sampling_rate` fields so log aggregators can compute accurate throughput from sampled data. Non-access loggers are never affected.
