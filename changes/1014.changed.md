The engine health-check server now defaults to binding loopback (`127.0.0.1`) instead of `0.0.0.0`, mirroring the Observatory hardening so liveness/readiness probes are not exposed off-host by accident. Set `[server.health] host = "0.0.0.0"` to restore off-host access. A new `[server.health] port_auto_increment` option (default `false`) lets an engine walk up from the configured port (8081, 8082, ...) when it is already taken, so several engines can run on one host without colliding on 8080. `protean upgrade-check` now flags the changed bind default.
