Projectors can now opt into **consume-side idempotency** with `@domain.projector(idempotent=True)`. Event delivery is at-least-once, so a redelivered event previously double-applied a non-idempotent projector (`total += 1`) and corrupted its read model. An idempotent projector records a `(message_id, handler)` marker in the same transaction as its read-model write, so a redelivery is applied exactly once on a transactional (relational) provider. Off by default; for cache-backed projections write idempotent upserts instead. See ADR-0017.
