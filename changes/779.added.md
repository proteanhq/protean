`protean check` gains two CI-standard output formats: `--format sarif` emits a SARIF 2.1.0 document (for GitHub Code Scanning upload and IDE problems panels), and `--format github-annotations` emits `::error`/`::warning`/`::notice` workflow-command annotations that surface findings inline on a PR without any SARIF-upload infrastructure. SARIF `reportingDescriptor` metadata is drawn from the enriched diagnostic schema (`rule.rationale`, `rule.fix`, `suggestion`); element locations resolve through the domain registry to the defining source file when available. `rich` remains the default and existing `rich`/`json` behaviour and exit codes are unchanged.
