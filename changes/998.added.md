CI now enforces the quality gates that previously only ran in pre-commit (and so were bypassable): a `ruff check` + `ruff format --check` lint job, and a project coverage floor (`coverage report --fail-under=94`) that fails a PR which drops overall coverage below 94%. Added a `dependency-review` supply-chain check on pull requests, extended Dependabot to the `github-actions` ecosystem, and added the community health files contributors and packagers look for — a `CODEOWNERS` file, a pull-request template, `CODE_OF_CONDUCT.md` (converted from `.rst`), and an `Issues` link in the PyPI project metadata.
