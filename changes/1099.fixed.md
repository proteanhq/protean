Fix an event-sourced `repository.get()` not returning the in-flight aggregate already cached in the current `UnitOfWork`. The identity-map lookup indexed the map with a flat identifier key, but the UnitOfWork stores it provider-nested (`{provider: {identifier: aggregate}}`), so the documented identity-map optimization never triggered — every `get()` re-loaded from the event store, and two `get()`s of the same aggregate within one UnitOfWork returned divergent objects. Surfaced by a mutation-testing pass over the event-sourcing path.
