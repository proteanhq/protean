Raise minimum runtime dependency versions: `typer` >= 0.26.8, `fastapi` >= 0.139.0, `uvicorn` >= 0.51.0, and OpenTelemetry >= 1.43.0 (`opentelemetry-api`/`opentelemetry-sdk`/`opentelemetry-exporter-otlp-proto-grpc`; `opentelemetry-instrumentation-fastapi`/`opentelemetry-exporter-prometheus` >= 0.64b0). As part of the OpenTelemetry bump, `TraceParent` now masks the W3C trace-flags byte (`0x01`) instead of comparing it literally, so `traceparent` headers carrying the new `RANDOM_TRACE_ID` flag (e.g. `03`, emitted by OpenTelemetry >= 1.43) are correctly read as sampled.
