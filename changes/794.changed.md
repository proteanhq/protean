Surface connection pool configuration in domain.toml with production-sensible defaults. SQLAlchemy pool defaults raised to pool_size=5 and max_overflow=10 for PostgreSQL and MSSQL. Redis broker, Redis cache, and MessageDB event store now forward pool parameters (max_connections, socket_timeout, etc.) from conn_info. A LOW_POOL_SIZE warning is emitted during domain validation when pool_size is set below 5.
