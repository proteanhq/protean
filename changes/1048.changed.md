**Behavioral change (synchronous processing):** a nested `current_domain.process(cmd, asynchronous=False)` call now returns *before* its downstream event cascade runs, instead of after. The final state once a chain settles is unchanged; only mid-handler visibility of downstream effects differs. If a handler dispatched a command synchronously and then read that command's *downstream* effects within the same handler, move that read to after the top-level call returns. See ADR-0016 Upgrade Notes.
