Opt-in lenient deserialization for legacy event/command payloads. Set the `lenient_deserialization` config key (or the per-event `lenient` meta option) so that deserializing a stored payload drops fields no longer on the current class — recording them under `metadata.extensions["_dropped_fields"]` — instead of raising a `DeserializationError`. The default stays strict (`extra="forbid"`): unknown fields raise unless lenience is explicitly enabled. Field-rename (`renamed_from`) aliases resolve first, so a renamed old key is kept, not dropped.
