Fixed a silent lost update in the SQLAlchemy adapter under optimistic concurrency. The version check in `_update` was a non-atomic read-compare-write, so under `READ COMMITTED` (PostgreSQL's default isolation level) two concurrent transactions could both pass the check and both write, losing an update with no `ExpectedVersionError`. Updates now issue a single conditional `UPDATE … WHERE _version = :expected` and raise `ExpectedVersionError` when the row's version has moved.
