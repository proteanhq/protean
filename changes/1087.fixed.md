Fixed a silent lost update in the SQLAlchemy adapter under optimistic concurrency. The aggregate version check was a non-atomic read-compare-write, so two concurrent transactions could both pass the check and both write, losing an update with no `ExpectedVersionError`. Aggregate models now carry a SQLAlchemy `version_id_col`, so the update flush is guarded by `WHERE _version = :expected` and a stale write raises `ExpectedVersionError`.
