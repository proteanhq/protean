Honor the `version=` option on `@domain.event` / `@domain.command` (and `domain.register(..., version=N)`) for classes that explicitly subclass `BaseEvent`/`BaseCommand` — previously the option was silently ignored on that path, leaving the schema version (and the `vN` type string) stuck at `1`. The version can be declared with either `version=` or the `__version__` class attribute; declaring it both ways on the same class raises `IncorrectUsageError`. Both forms now validate the version as a positive integer and reject booleans — note that `__version__ = True`, previously accepted and silently treated as version `1`, now raises `IncorrectUsageError` at class definition.
