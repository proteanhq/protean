Element option defaults (`_default_options`) are now a declarative `ClassVar` data attribute on each element root instead of a classmethod evaluated at class-definition time; defaults that derive from the class (e.g. a schema or stream name) use a new `DerivedDefault` marker resolved when options are assembled. This is an internal (private) API change with no user-facing behaviour difference — it removes class-definition-time method execution and lets tooling (e.g. mutation testing) instrument element modules cleanly.
