The built-in email subsystem is deprecated and will be removed in v1.0.0; sending email is an infrastructure concern that belongs in an application-level notification service rather than the domain framework. Every email entry point still works but now surfaces a `DeprecationWarning` (`RemovedInProtean10Warning`): registering an email element (via `@domain.email` or `domain.register(...)`), calling `domain.send_email(...)` or `domain.get_email_provider(...)`, and configuring a non-default `[email_providers]` block (a default, untouched config stays silent). Run Python with `-W default::DeprecationWarning` to see them, and `protean check` now reports each registered email element as a `DEPRECATED_EMAIL` info diagnostic (it does not fail the check). Migrate by handling the domain event from an event handler or subscriber that calls your own notification service; see the [v0.17 migration guide](https://docs.proteanhq.com/reference/migration/v0-17/).
