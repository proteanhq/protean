`protean check` now reports `UPCASTER_GAP`: an event whose `__version__` is N>1 but has no upcaster path from a prior version (1..N-1) to N is flagged at build time, instead of failing only at read time with a `DeserializationError` when an old stored payload is loaded. Both a total absence of upcasters and partial coverage (e.g. a v3 event with only a 2→3 upcaster leaves v1 stranded) are caught. Relatedly, an upcaster's `event_type` may now be given as a string forward reference: it resolves by name to the registered event, and a string naming an unregistered event surfaces as a clean structured error.
