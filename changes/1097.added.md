Mutation testing for Protean core: a `make mutation` target (and `scripts/mutation.sh`) that mutates a module and checks whether its fast unit-test subset notices, reporting surviving mutants (executed-but-unasserted code paths). It runs in a dedicated Python 3.12 environment so the project environment is untouched. The first pass hardened the outbox test suite (`src/protean/utils/outbox.py`), closing genuine gaps around retry/lock boundary instants, query limits, the `target_broker` filter, and the reconciliation tail scan. See `docs/community/contributing/mutation-testing.md`.
