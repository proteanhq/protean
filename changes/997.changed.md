The entire `src/protean` codebase now passes `mypy --strict` (global `strict = true`, the type-safety quarantine emptied) and `pyright` in standard mode — a dual gate: mypy for bug-catching, pyright for editor/Pylance fidelity (configured in `pyrightconfig.json`). Every public API is annotated and the `py.typed` marker is fully honored, so downstream projects get precise type information and editor completions. Adopting strict typing also surfaced and fixed ~22 latent runtime bugs, including: `python -m protean` was broken (raised `TypeError` instead of running the CLI); `ProteanException` string rendering crashed on non-dict messages; a `TypeError` in the SQLAlchemy provider's `_raw`; a `None`-dereference in the Elasticsearch adapter's `_update`; a `KeyError`-contract violation in the `Providers`/`Caches` containers; several unguarded `Optional` event-store accesses; `MemoryCache.flush_all` breaking `set_ttl`/`get_ttl`; `Node.__eq__` raising on unrelated objects; and instance-vs-class annotation mistakes across registry/handler lookups. See ADR-0018 for the strategy (quarantine-then-strictify, plus pyright for editor fidelity).
