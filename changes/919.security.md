Mask sensitive fields in log output, serialize multi-worker log streams, and emit domain-boundary invariant/validation failures to a dedicated `protean.security` logger. New `ProteanRedactionFilter` and `make_redaction_processor()` wire into `Domain.configure_logging()` via `[logging].redact` (defaults: `password`, `token`, `secret`, `api_key`, `authorization`, `cookie`, `session`, `csrf`); `log_method_call` automatically inherits redaction. `protean server --workers N` (N > 1) now installs a `QueueHandler`/`QueueListener` pattern so JSON log lines never interleave across processes. `InvalidOperationError`, `InvalidStateError`, aggregate invariant violations, and boundary-crossing command validation errors emit WARNING events on `protean.security` for SIEM/alerting integration.
