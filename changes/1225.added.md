`protean check` gains the opt-in `ADAPTER_CALL_IN_DOMAIN` rule: it flags a domain element whose method body calls a concrete adapter under `protean.adapters`, catching runtime coupling one level deeper than `INFRA_IMPORT_IN_DOMAIN` (which reads only module-level imports). It is off by default, enabled with `[lint].check_adapter_calls = true`, and conservative by construction — a call whose callee does not statically resolve is skipped, never guessed at.
