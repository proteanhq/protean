Unified the two `deprecated` option normalizers into a single implementation, `protean.utils._normalize_deprecated`. Field-level construction (`Field.__init__` and `FieldSpec.__init__`) now delegates to the same normalizer already used by the `@domain.<element>(deprecated=...)` decorator path, instead of a duplicate `protean.fields.base.normalize_field_deprecated`. **Behavioral change:** invalid `deprecated` values passed to a field (e.g. a dict missing `since`, or `deprecated=True`) now raise `ConfigurationError` instead of `ValueError`, matching the element-level behavior. `deprecated=False` continues to be tolerated as an unadvertised alias for `None` at both field and element level.
