Dropped the standalone `elasticsearch-dsl` dependency from the `elasticsearch` extra and raised the `elasticsearch` floor to `>=8.18.0` (still `<9.0.0`). The DSL layer was folded into the `elasticsearch` package itself as of 8.18, so the adapter now imports it from `elasticsearch.dsl`; the previous `>=7.17.9` floor allowed versions where that module does not exist. Projects on the `elasticsearch` extra should install `elasticsearch>=8.18.0` and, if they imported the DSL directly, switch `import elasticsearch_dsl` to `from elasticsearch import dsl`.
