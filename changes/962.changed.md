Support the Elasticsearch 8.x client (`elasticsearch` / `elasticsearch-dsl` 8.x), which installs now resolve to by default. The provider normalizes bare host entries to `scheme://host:port` URLs (using `USE_SSL` to choose the scheme) and no longer passes the v7-only `use_ssl` flag, so existing host configuration keeps working. This also allows `urllib3` 2.x, resolving the related security advisory. **The 8.x client requires an Elasticsearch 8.x server.** Deployments still running an Elasticsearch 7.x server should pin `elasticsearch<8` to keep the 7.17 client, which connects to both 7.x and 8.x servers. See the [v0.16 migration guide](https://docs.proteanhq.com/reference/migration/v0-16/) for details.
