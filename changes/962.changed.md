Support the Elasticsearch 8.x client (`elasticsearch` / `elasticsearch-dsl` 8.x) while remaining compatible with 7.17.x. The provider now normalizes bare host entries to `scheme://host:port` URLs (using `USE_SSL` to choose the scheme) and no longer passes the v7-only `use_ssl` flag, so existing host configuration keeps working on both client versions. This also resolves a `urllib3` security advisory by allowing `urllib3` 2.x. The adapter is validated against both the 7.17.x and 8.x clients.
