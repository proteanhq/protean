Multi-step process managers now cascade to completion under `event_processing = "sync"`. Synchronous event dispatch is now breadth-first, so each handler's UnitOfWork (including a process manager's transition) commits before the next event is processed — a saga no longer stalls after its start handler, and a projector for an originating event runs before a projector for a nested event (no more `ObjectNotFoundError` on create-then-update read models). See ADR-0016.
