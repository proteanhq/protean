Emit one wide event per HTTP request on the `protean.access.http` logger when FastAPI apps use `DomainContextMiddleware`. The event carries the request envelope (method, path, status, duration), route name/pattern, commands dispatched during the request, client IP, user agent, and both `request_id` (from `X-Request-ID` or auto-generated) and `correlation_id`. Level is INFO for 2xx/3xx, WARNING for 4xx, ERROR for 5xx. Responses always echo `X-Request-ID` so operators can paste it into log aggregators to find the matching wide event. New `[logging.http]` config section (`enabled`, `exclude_paths`, `log_request_headers`, `log_response_headers`) lets operators tune HTTP wide events without touching Python. Application code enriches the event via the same `bind_event_context()` API as domain wide events — bindings flow through the async task boundary and land on both the HTTP and domain layers with the shared `correlation_id`.
