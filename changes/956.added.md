Add the `UNBOUNDED_INDEXED_STRING` build-time lint rule (category `persistence`, warning level): `protean check` now flags any aggregate field that is both included in a declared index and an unbounded string (`Text`, or `String(max_length=None)`). Such an index produces DDL that fails on SQL Server, needs an explicit prefix length on MySQL, and carries storage and performance overhead on PostgreSQL. Give the field a bounded `max_length`, or remove it from the index.
