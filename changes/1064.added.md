Added a `--canonical` flag to `protean ir show` that omits the volatile `generated_at` timestamp, producing a stable baseline for committed `.protean/ir.json` snapshots. The `--fix` staleness pre-commit hook and `protean schema generate` now write the same canonical output, so regenerating a baseline yields a git diff only when the domain contract actually changes. `generated_at` is now optional in the IR schema, keeping canonical baselines schema-conformant. Upgrading produces a one-time diff on the next regeneration (dropped timestamp, sorted keys); staleness and diff checks are unaffected.
